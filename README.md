# Посуточная аренда квартир — лендинг

Статический одностраничный сайт без сборки: `index.html`, `styles.css`, `app.js` и папка `data/` с JSON. Деплоится на GitHub Pages прямо из ветки `main` (без GitHub Actions). Только vanilla JS, никаких фреймворков.

## Что где править

### 1. Контакты и название
Всё в начале файла **`app.js`** — блок «КОНФИГУРАЦИЯ»:

```js
const BRAND_NAME = "Уютные квартиры";       // название бизнеса
const CITY = "Москва";                       // город / район
const PHONE = "+7 900 000-00-00";           // телефон (кнопка «Позвонить»)
const TELEGRAM_USERNAME = "your_username";   // username в Telegram без @
```

Поменяли значения → сохранили → закоммитили → запушили. Сайт обновится автоматически.

### 2. Квартиры
Файл **`data/apartments.json`** — массив из трёх (или сколько нужно) объектов:

```json
{
  "id": "studio-center",
  "title": "Уютная студия в центре",
  "pricePerNight": 2500,
  "description": "Короткое описание квартиры…",
  "amenities": ["Wi-Fi", "Кондиционер", "Стиральная машина"],
  "photos": ["assets/studio-1.jpg", "assets/studio-2.jpg"]
}
```

- `id` — латиницей, уникальный (используется для привязки календаря и занятости).
- `photos` — список путей к фото в папке `assets/`. Если пусто `[]`, в карточке показывается заглушка «Фото скоро добавим».

### 3. Фото квартир
Кладёте файлы в папку **`assets/`** (например `studio-1.jpg`) и прописываете их имена в `photos` нужной квартиры. Несколько фото → в карточке появляется галерея со стрелками.

### 3.1. Что рядом (достопримечательности)
Файл **`data/landmarks.json`** — массив карточек с фото города:

```json
{
  "title": "Памятник затопленным кораблям",
  "photo": "assets/landmarks/sunken-ships.jpg",
  "description": "Короткая подпись…",
  "credit": "Автор · Лицензия"
}
```

Фото лежат в `assets/landmarks/`. Сейчас стоят виды центра Севастополя из Wikimedia Commons (свободные лицензии, авторство указано в поле `credit` и выводится мелким текстом на карточке). Можете заменить на свои.

### 4. Отзывы
Файл **`data/reviews.json`** — массив:

```json
{ "name": "Ольга", "rating": 5, "text": "Текст отзыва…" }
```

`rating` — от 1 до 5, рисуется звёздами.

### 5. Занятые даты
Файл **`data/availability.json`** — по `id` квартиры массив занятых дат и/или диапазонов:

```json
{
  "studio-center": [
    "2026-06-10",
    { "from": "2026-06-15", "to": "2026-06-18" }
  ]
}
```

- Одиночная дата — строкой `"YYYY-MM-DD"`.
- Диапазон (включительно) — объектом `{ "from": "…", "to": "…" }`.

Занятые даты в календаре — серые и перечёркнутые, свободные — зелёные и кликабельные (по клику показывается подсказка «Свободно — позвоните или напишите»).

## Локальный просмотр

Сайт грузит JSON через `fetch`, поэтому просто открыть `index.html` двойным кликом не получится (CORS для `file://`). Запустите локальный сервер:

```bash
# Python 3
python -m http.server 8000
# затем откройте http://localhost:8000
```

## Будущее: авто-синхронизация занятости с Авито (Supabase + iCal)

Сейчас занятость берётся из локального `data/availability.json`. В коде это инкапсулировано в одной функции в `app.js`:

```js
async function getAvailability(apartmentId) {
  // сейчас: локальный JSON
}
```

Когда захотите автоматически подтягивать занятые даты из календаря Авито — менять нужно **только эту функцию**, остальной код останется как есть.

### План подключения

1. **Авито → iCal.** В личном кабинете Авито у каждого объявления есть ссылка-экспорт календаря броней в формате iCal (`.ics`). Скопируйте ссылку для каждой квартиры.

2. **Supabase: таблица.** Создайте таблицу `availability`:
   - `apartment_id` (text)
   - `date` (date)  — одна строка = один занятый день
   - (опц.) `source` (text), `updated_at` (timestamptz)

3. **Supabase: крон-функция.** Edge Function (или pg_cron + сервис), которая раз в N часов:
   - ходит по iCal-ссылкам Авито каждой квартиры,
   - парсит `VEVENT` (даты `DTSTART`/`DTEND`),
   - разворачивает их в отдельные дни,
   - перезаписывает строки в таблице `availability` для этой квартиры.

4. **Фронт: переключить источник.** Замените тело `getAvailability` на запрос к Supabase REST:

   ```js
   const SUPABASE_URL = "https://<project>.supabase.co";
   const SUPABASE_ANON_KEY = "<anon-key>"; // публичный ключ, только чтение

   async function getAvailability(apartmentId) {
     const res = await fetch(
       `${SUPABASE_URL}/rest/v1/availability?apartment_id=eq.${apartmentId}&select=date`,
       { headers: { apikey: SUPABASE_ANON_KEY, Authorization: `Bearer ${SUPABASE_ANON_KEY}` } }
     );
     const rows = await res.json();
     return rows.map(r => r.date); // вернёт массив "YYYY-MM-DD" — формат тот же
   }
   ```

   Включите RLS и политику «только SELECT» для анонимной роли — ключ публичный, но доступ только на чтение.

Формат, который возвращает `getAvailability` (массив дат-строк и/или диапазонов), остаётся прежним, поэтому календарь продолжит работать без изменений.
